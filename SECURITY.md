# Security Best Practices for AR Chem Lab

To maintain the security of the application and protect user data and sensitive credentials, please follow these guidelines, especially when preparing for a production release.

## 1. Secrets Management

The application uses the `flutter_dotenv` package to securely load environment variables.

- **NEVER** commit the `.env` file to version control. It is already added to `.gitignore`.
- When building the app via a CI/CD pipeline, ensure that the `.env` file is generated or injected securely using pipeline secrets before running `flutter build`.

Current variables in use:

- `GEMINI_API_KEY`: The API key used for the Gemini Chat feature.

## 2. Local Storage

We use `shared_preferences` for local storage in the application. We have conducted a security review and confirmed that only non-sensitive user preferences are stored (e.g., onboarding completion status, list of most viewed elements).

- Do not store sensitive information like passwords, API keys, or personal identifiable information (PII) in `shared_preferences`. If such need arises in the future, use `flutter_secure_storage` instead.

## 3. Code Obfuscation

To protect the source code from reverse engineering, it's highly recommended to use code obfuscation when building the application for production.

To build an obfuscated APK, use the following command:

```bash
flutter build apk --obfuscate --split-debug-info=build/app/outputs/symbols
```

To build an obfuscated App Bundle (for Google Play Store):

```bash
flutter build appbundle --obfuscate --split-debug-info=build/app/outputs/symbols
```

**Note**: Save the `symbols` directory generated by these commands. You will need it to de-obfuscate any crash reports you receive from the production app.
